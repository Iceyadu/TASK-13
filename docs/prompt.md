Implement a HarborView Property Operations Portal for HOA and multifamily teams to manage resident billing, community listings, and service orders in a single offline-capable site. The Vue.js front end runs as a responsive web app with an optional installable offline mode so property staff can work from a leasing office desktop while residents can use tablets or shared kiosk devices on the building network. Users sign in with a local username and password, then see role-based navigation for Administrator (system setup), Property Manager (operations), Accounting Clerk (billing and reconciliation), Maintenance/Dispatcher (order fulfillment), and Resident (self-service). Residents can maintain a shipping/mailing address, view and download statements, submit payment evidence as a scanned image (for check or money order), request refunds as credits, and track service orders through clear status badges and timestamped milestones. Property staff can create and manage marketplace-style listings (e.g., garage sale item, parking spot sublet, amenity booking add-on) with draft saving, edit and publish/unpublish controls, bulk status changes, and embedded media uploads with immediate validation feedback; images are limited to JPG/PNG up to 10 MB each, videos to MP4 up to 200 MB, and the UI flags unsupported formats before saving. The homepage and site content are configurable by admins using modular sections such as carousel panels, recommended tiles, and announcement banners, with preview mode and a controlled rollout toggle so a new configuration can be shown to 10% of staff accounts first before full enablement.

The backend uses FastAPI to expose RESTful APIs consumed by the Vue.js client, with PostgreSQL as the system of record for residents, properties, fee rules, bills, listings, orders, content, and audit logs. All workflows are designed for offline fault tolerance: the client maintains an encrypted local cache and a retry queue for writes, automatically replaying requests when the app reconnects to the on-prem server; every writable record includes a version number for optimistic locking so conflicting edits are detected, presented to the user with a side-by-side diff, and resolved via keep mine, keep theirs, or merge fields rules. Orders follow a strict state machine (created, payment recorded, accepted, dispatched, arrived, in-service, completed, after-sales credit) with key-node validation, idempotency keys to prevent double submissions during retries, and immutable operation logs for audit traceability. Billing supports configurable fee items and cycle rules (e.g., monthly on the 1st, late after 10 days, $25.00 late fee, 6.00% sales tax only on taxable line items), automated bill generation per property, reconciliation against recorded payments, refund handling as credit memos, overdue reminders displayed in-app, locally generated PDF receipts, and CSV export for financial reporting. Security is local-first: passwords require at least 12 characters with complexity checks and are stored with strong hashing; sensitive fields such as SSNs (if collected) are disallowed by default, while resident contact data is encrypted at rest and masked in screens by role. Nightly encrypted backups are written to a designated local drive with a 30-day retention and a restore workflow that can be executed without any internet dependency.
